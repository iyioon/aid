## Task Analysis

First, carefully read and understand the task requirements provided in $ARGUMENTS.

## Step-by-Step Execution

1. **Explore the Codebase**
   - Identify relevant files and code patterns
   - Understand the existing architecture
   - Note dependencies and related functionality

2. **Plan the Implementation**
   - Break down the task into smaller steps
   - Identify files to create or modify
   - Consider edge cases and error handling

3. **Implement Changes**
   - Make changes incrementally
   - Follow existing code style and patterns
   - Add appropriate error handling

4. **Commit Progress**
   - After each logical unit of work, commit with a clear message
   - Use conventional commit format (feat:, fix:, docs:, etc.)

5. **Continue Until Complete**
   - Work through all planned steps
   - Ensure the implementation is complete and functional

When implementation is complete, run `/review-work` to self-review before creating a PR.
